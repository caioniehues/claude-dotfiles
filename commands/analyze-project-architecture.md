<architecture_thinking>

## Error Recovery Thinking
<error_thinking>
When analysis encounters issues:
1. **No clear entry point found**:
   - Thinking: What alternative indicators exist?
   - Recovery: Look for build files, common directories

2. **Technology stack unclear**:
   - Thinking: What file patterns can guide me?
   - Recovery: Search for language/framework indicators

3. **Complex/unusual structure**:
   - Thinking: What conventions might they be following?
   - Recovery: Focus on most informative areas first

4. **Analysis scope too large**:
   - Thinking: What's the most valuable subset to analyze?
   - Recovery: Create todo list, prioritize by user needs
</error_thinking>
</adaptive_error_handling>
